> 目标：让评委只看材料/代码/演示，就能快速验证“确实深度使用高德能力 + 作品流程完整 + 有创新彩蛋”。

## 高德 API 调用清单（核心证据）

### Web App（前端）
- **高德 JS API（地图展示）**
  - 用途：地图底图、Marker、Polyline 绘制、视野自适应
  - 配置：`VITE_AMAP_JS_KEY`

### 服务端（推荐主流程）
- **POI 周边检索**：`/v3/place/around`
  - 用途：按意图关键词拉取候选 POI（按距离排序）
  - 证据：`api/recommend.ts` / `server/src/index.ts` 中 `amapPlaceAround`
- **路径规划**
  - 步行：`/v3/direction/walking`
  - 驾车：`/v3/direction/driving`
  - 骑行：`/v4/direction/bicycling`
  - 用途：对 Top1 精算“去/回”时间，产出 polyline
  - 证据：`api/recommend.ts` / `server/src/index.ts` 中 `amapDirection` + `extractDurationAndPolyline`
- **静态地图**：`/v3/staticmap`
  - 用途：生成路线概览图（可嵌入报告/演示）
  - 证据：`api/staticmap.ts` / `server/src/index.ts`（同名路由）

### 彩蛋（挑战/虚拟宝藏）
- **定位能力（浏览器定位 / 可替换为高德定位 SDK）**
  - 用途：到达验收（用户定位进入 POI 周边半径即可）
  - 证据：前端 `src/App.tsx` 的 `locate()` 与 `/api/egg-verify`

## 技术架构说明（评委快速理解）

### 主流程（“随机地点打发时间”）
1. 前端收集：起点 + 时间段 + 交通方式 + 一句话偏好
2. 后端：调用高德周边检索拉候选 POI
3. 后端：意图解析（**AI 主导 + 规则兜底**），并做“意图-类别亲和度”重排
4. 后端：对 Top1 调用路径规划精算去/回时间，保证“时间闭环”
5. 前端：展示结果 + 一键跳转高德导航

### 彩蛋（“开启彩蛋”）
1. 前端在结果卡点击“开启彩蛋”
2. 后端：检查用户选择的时间段是否为白天（安全）
3. 后端：生成“挑战故事 + 安全任务”
4. 前端：点击“我已到达”，调用 `/api/egg-verify` 完成半径校验

## 创新点清单（简洁可讲）
- **时间闭环决策**：去/玩/回预算强约束，专为碎片时间设计

- **彩蛋玩法**：低社交、白天安全触发的“碎片时间挑战/虚拟宝藏”
